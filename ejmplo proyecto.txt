--------------------Vamos a hablar de dataBase

                    En Node.js existen dos librerías populares para conectarse a bases de datos MySQL:

mysql → la librería más vieja.

mysql2 → una versión mejorada, más rápida y más actualizada.

-------------------Ventajas de mysql2:

                    Soporta Promises y async/await (lo cual facilita el código moderno).

                    Es más rápido y eficiente.

                    Está mejor mantenido que mysql.

                    Por eso, hoy en día se recomienda usar mysql2 en lugar de mysql.


--------------------require  que es ? 

                En Node.js, cuando queremos usar una librería externa (como mysql2), debemos importarla en nuestro archivo.
const mysql = require('mysql2');

                "Traeme el paquete mysql2 que instalamos con npm install mysql2 y
                 guardalo dentro de la constante mysql para poder usarlo en el código".  


--------------------------------porque ahora es const, require  en vez de import .
--------------------------------al igual que en vez de export default ahora es  module.exports

                    Esto tiene que ver con el sistema de módulos que usa Node.js.

        require y module.exports → pertenecen a CommonJS (el sistema de módulos original de Node.js).

        import y export default → pertenecen a ES Modules (ESM), que es el estándar de JavaScript más moderno.

En CommonJS (por defecto en Node.js):

const mysql = require('mysql2');   // importar
module.exports = { connection };   // exportar

En ES Modules 

import mysql from 'mysql2';        // importar
export default connection;         // exportar

pero para ello debevamos activar "type": "module" en package.json):

------------------------------------¿Qué hace module.exports = { connection }?

                        Esto significa:

                        "Voy a exportar este objeto { connection } para que pueda ser usado en otros archivos de mi proyecto".

                        Entonces, en otro archivo podriamos hacer:

                        const { connection } = require('./database');


                        Y ya tenemos la conexión lista para usar.



--------------------------------------------------------------------Routes----------------------------------------------

Importamos Express   const express = require('express');

Express es un framework de Node.js que nos facilita crear un servidor y manejar rutas HTTP (GET, POST, PUT, DELETE).

En vez de escribir todo a mano con Node puro, Express nos da funciones ya listas.

------------------------Creamos un router  const router = express.Router();

                        Router() es como un mini servidor dentro de Express.

Sirve para organizar las rutas en archivos separados, en lugar de tener todo en index.js.

Ejemplo: podemos tener un usuarios.routes.js, otro clientes.routes.js, etc. 


   Importaciones 
                const { 
    mostrarUsuarios,
    mostrarUsuario,                                             
    crearUsuario,                       
    editarUsuario,
    eliminarUsuario 
} = require('../controllers/usuarios');                                 


---------------Estas funciones están en otro archivo (controllers/usuarios.js).

Cada una representa la lógica que se ejecuta cuando un usuario hace una petición.


mostrarUsuarios → devuelve todos los usuarios.

mostrarUsuario → devuelve un usuario según su id.

crearUsuario → agrega un nuevo usuario a la base de datos.

editarUsuario → actualiza un usuario existente.

eliminarUsuario → borra un usuario.



Definimos las rutas

router.get("/usuarios", mostrarUsuarios);     // Obtener todos los usuarios
router.get("/usuario/:id", mostrarUsuario);   // Obtener un usuario por id
router.post("/usuario", crearUsuario);        // Crear un nuevo usuario
router.put("/usuario/:id", editarUsuario);    // Editar un usuario existente
router.delete("/usuario/:id", eliminarUsuario); // Eliminar un usuario

El :id significa un parámetro dinámico en la URL. Ejemplo:

/usuario/5 → obtiene o edita o elimina el usuario con id = 5.


Las rutas en Express sirven para definir cómo nuestro backend responde a las peticiones HTTP que vienen desde el frontend o desde cualquier cliente (por ejemplo, Postman).

En tu ejemplo, estas rutas son parte de un CRUD de usuarios (Crear, Leer, Actualizar y Borrar).

   "Las rutas son como caminos que el frontend o cualquier cliente pueden usar para hablar con el servidor.
   
   "
